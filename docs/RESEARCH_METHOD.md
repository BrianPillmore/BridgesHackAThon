# Research and idea scoring method

## Scope

The library focuses on present-day problems where a municipality or public-school
system has a clear operating role and a small web application can demonstrate
value without pretending to solve the entire policy problem.

## Source hierarchy

1. Current federal, state, or local government data/guidance.
2. Official open-data portals and oversight reports.
3. Primary research or established nonpartisan research institutions.
4. Practitioner reporting only when primary evidence is unavailable.

Facts in the briefs link to the source. Data dates are stated where material.
Older data are retained only when they define scale or a durable operational need.

## Scoring

Each idea receives five 1–5 scores:

- **Public value:** plausible benefit to safety, access, learning, trust, or capacity.
- **One-hour feasibility:** convincing vertical slice with the included stack.
- **Data readiness:** public, synthetic, or easily supplied input exists.
- **Adoption clarity:** a specific user and workflow can own the tool.
- **Privacy/safety tractability:** risk can be bounded in a demo and reviewed in a pilot.

Weighted score:

```text
30% public value
25% one-hour feasibility
20% data readiness
15% adoption clarity
10% privacy/safety tractability
```

Scores are comparative planning judgments, not evidence that an intervention will
work. A selected idea still needs user interviews and a pilot hypothesis.

## Inclusion test

An idea is included only when its brief can specify:

- named primary user;
- current workflow pain;
- a five-step demo;
- a data source or synthetic fallback;
- measurable outcome;
- explicit safeguards and non-goals.
