# Master plan

## Outcome

Demonstrate one evidence-backed municipal or public-education workflow on a live
Firebase App Hosting URL within one hour, then preserve a credible path to a pilot.

## Phases

### Before the conference

- Validate tooling, lock dependencies, create backend, rehearse deployment.
- Review `brainstorm/SUMMARY.md` and pre-select two likely ideas.
- Identify one public-data sample and one synthetic fallback per finalist.

### First 10 minutes

- Select idea and owner.
- Lock user, decision, outcome, and cut line.
- Create `plans/current-idea.md`.

### Build

- Complete golden path by minute 25.
- Make value legible by minute 35.
- Add resilient states and tests by minute 45.

### Launch

- Preflight, deploy, health check, mobile check, rehearse.

### After judging

- Conduct user interviews.
- Validate outcome metric and workflow ownership.
- Complete privacy/security/accessibility review before real data.
- Decide stop, revise, or pilot.
