# Ninety-second demo script

## 0:00–0:15 — problem

“[Primary user] currently has to [painful workflow]. [Evidence/stat] shows why this
matters. The hard decision is [decision].”

## 0:15–0:30 — before

“Today the input looks like this: [noise, fragmentation, delay, language barrier,
or missing status].”

## 0:30–1:10 — product

“Here is the same workflow in the prototype.”

1. Show the prepared input.
2. Trigger the transformation.
3. Point to the ranked/summarized result and explanation.
4. Take the human action or correction.
5. Show the visible outcome.

## 1:10–1:25 — value

“This could reduce/improve [metric]. The first pilot would compare [baseline] with
[prototype outcome] for [users/time period].”

## 1:25–1:30 — boundary and ask

“This does not automate [consequential decision] and uses [public/synthetic] data.
Our next step is [specific user test/pilot request].”
